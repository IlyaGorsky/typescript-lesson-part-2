# Глоссарий TypeScript — Часть 2

| Термин | Описание |
| ------ | -------- |
| **Interface** | Именованный контракт, описывающий форму объекта. Поддерживает наследование (`extends`) и слияние деклараций (declaration merging) |
| **Declaration Merging** | Автоматическое объединение нескольких объявлений `interface` с одинаковым именем в одно |
| **Union Type** | Тип-объединение (`A \| B`) — значение может быть одним из перечисленных типов |
| **Intersection Type** | Тип-пересечение (`A & B`) — значение должно удовлетворять всем перечисленным типам одновременно |
| **Narrowing** | Сужение типа — компилятор уточняет тип переменной внутри условных блоков (`if`, `switch`, `in`, `typeof`, `instanceof`) |
| **Discriminated Union** | Union-тип, где каждый вариант имеет общее поле-дискриминант (например, `type: "success" \| "error"`), позволяющее сужать тип через `switch` |
| **Exhaustiveness Check** | Проверка полноты обработки всех вариантов union через тип `never` — компилятор ругается, если добавлен новый вариант, но не обработан |
| **Type Guard** | Функция-предикат с возвращаемым типом `value is Type`, которая сужает тип в условных блоках |
| **Function Overload** | Перегрузка функций — несколько сигнатур для одной функции, позволяющие возвращать разные типы в зависимости от аргументов |
| **Generics** | Обобщённые типы (`<T>`) — параметризация типов, позволяющая писать универсальный код с сохранением типобезопасности |
| **as const** | Assertion, который делает значение максимально узким: литеральные типы, `readonly` массивы и объекты |
| **satisfies** | Оператор, проверяющий соответствие значения типу без расширения — сохраняет литеральные типы |
| **Literal Type** | Конкретное значение как тип (`"hello"`, `42`, `true`), а не обобщённый `string`, `number`, `boolean` |
| **Mapped Type** | Тип, построенный перебором ключей: `{ [K in Union]: Type }`. Так работают `Pick`, `Readonly` и другие utility types |
| **keyof** | Оператор, возвращающий union всех ключей объектного типа (`keyof { a: 1; b: 2 }` → `"a" \| "b"`) |
| **typeof** | Оператор уровня типов — извлекает тип из значения (`typeof myVar`) |
| **Pick<T, K>** | Utility type — создаёт тип из выбранных ключей `K` типа `T` |
| **Omit<T, K>** | Utility type — создаёт тип из всех ключей `T`, кроме `K` |
| **Readonly<T>** | Utility type — делает все поля типа `T` доступными только для чтения |
| **ReturnType<T>** | Utility type — извлекает тип возвращаемого значения функции `T` |
| **Branded Type** | Номинальная типизация — добавление «метки» к примитивному типу, чтобы `UserId` и `TodoId` были несовместимы, хотя оба `number` |
| **Declaration File (.d.ts)** | Файл, содержащий только описания типов без реализации — позволяет использовать JS-библиотеки в TypeScript |
